What Does 1 in 48 Mean?

Out of every 48 predictions the model makes on unseen data, on average only 1 is incorrect. The other 47 are correctly classified as either Malignant or Benign.

This is measured on the test set — patients the model had never seen during training, ensuring the number is a genuine reflection of real-world performance and not overfitting.

Why False Negatives Matter Most

Not all errors are equal. We deliberately tuned the model to minimise false negatives — cases where cancer is missed.

  • A false negative tells a patient their sample looks benign when it may actually be malignant. This is the most dangerous error — it may delay diagnosis and treatment.
  • A false positive tells a patient their sample looks malignant when it is actually benign. This causes unnecessary anxiety — but always leads to further medical consultation, which is safer.
  • Our class weighting is set to {Benign: 1, Malignant: 2} — the model is penalised twice as heavily for missing a malignant case as for a false positive. This is a deliberate ethical design decision.
  • The result: Our model leans toward flagging suspicious cases rather than dismissing them. This is the correct trade-off for a medical screening tool.

The Dataset

The model was trained on one of the most well-studied medical ML datasets in the world.

Wisconsin Breast Cancer Dataset

The Breast Cancer Wisconsin (Diagnostic) dataset, compiled by Dr. William H. Wolberg at the University of Wisconsin Hospitals, Madison. Widely used in medical ML research since 1993.

569 Patient Records

569 samples from real FNA biopsy procedures. 357 benign (62.7%) and 212 malignant (37.3%) — a realistic class distribution reflecting real clinical populations.

30 Original Features

The original dataset contains 30 computed features — 10 base measurements each recorded as mean, SE, and worst. We selected the 13 most predictive features after careful analysis.

75/25 Train-Test Split

426 samples used for training, 143 for testing. The test set is kept completely separate during training — the model never sees those patients until final evaluation.

Binary Classification

Each sample is labelled either Benign (B → 0) or Malignant (M → 1). The model outputs both a class prediction and a probability confidence score.

Class Imbalance Handling

The dataset has more benign than malignant samples. We handle this with class weighting rather than oversampling, preserving the natural data distribution.

How the Model Was Built

A full walkthrough of the machine learning pipeline — from raw data to deployed prediction.

1
Data Loading

Raw CSV loaded with pandas. ID column and 17 low-importance features dropped. Diagnosis encoded: B→0, M→1.

2
Feature Selection

13 of 30 features retained based on predictive importance. Low-variance and highly correlated features removed.

3
Train/Test Split

75% training (426 samples), 25% testing (143 samples). random_state=42 ensures reproducibility.

4
Normalisation

MinMaxScaler fitted on training data only, then applied to both sets. Scaler saved to prevent data leakage at inference.

5
Model Training

Random Forest with 300 trees. Class weight {0:1, 1:2} applied to penalise false negatives twice as heavily.

6
Serialisation

Model, scaler, and feature order all saved as .pkl files. Feature order is critical — ensures inputs are always aligned correctly.

Why Random Forest?

Random Forest was chosen deliberately over simpler or more complex alternatives for several reasons.

  • High accuracy on tabular data — Random Forests consistently outperform single decision trees and match or beat many neural networks on structured clinical datasets of this size.
  • Robust to overfitting — By averaging 300 diverse trees, the ensemble is far less likely to memorise training data than a single deep tree.
  • Handles feature correlation — Breast cancer features like radius, perimeter, and area are highly correlated. Random Forest's random feature selection at each split handles this gracefully.
  • Native probability output — predict_proba() gives a confidence score (0–100%) for each prediction, adding transparency to the result.
  • No need for feature scaling for the trees — though we still normalise for consistency and to make the scaler reusable at inference time.
  • Interpretable feature importance — We can identify which features the model relies on most, adding clinical interpretability.

Ethical Design Decisions

Every technical choice in this project was made with patient safety and responsible AI in mind.

Class Weighting {Benign:1, Malignant:2} The model is explicitly penalised twice as heavily for missing a malignant case. Missing cancer is far worse than a false alarm — so the model is tuned to err on the side of caution.

Persistent Disclaimer on Every Result Every prediction — including a Benign result — is accompanied by a clear disclaimer that the output is not a medical diagnosis and professional consultation is essential.

No Data Storage No patient inputs are logged, stored, or transmitted. Each session is fully stateless. Users' medical information stays private.

Confidence Score Transparency The model doesn't just say "Malignant" — it tells you how confident it is. A 55% confidence result should be treated very differently to a 98% confidence result.

Honest Limitation Communication The site clearly explains that these features require a clinical FNA biopsy — not something a user can self-measure. We never pretend this tool is more accessible than it is.

Open Source & Reproducible The full model training code, dataset reference, and feature selection process are documented and available. Nothing is a black box.
